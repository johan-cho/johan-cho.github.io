portfolio site

this site!



I made a portfolio site for a class assignment (encp3000), but it didn't look great and was slow, unintuivite and hard to read. This is my attempt at a better one that considers the user experience, while also appearing "clean".

It took a while; all of the html, css and js is vanilla, a lot of that either implements things that have already been done by frameworks or queries the github api (all of the language stats per repo and overall are realtime).

One thing that I need to think about is how do I make the language bar a little more intuivite? Most users don't understand what it means, and that you could hover over it to reveal the languages and their percentages.

Feel free to send me some feedback; UX is a continuous process that can only improved over time.
